# v1.14.0 Release Notes

`v1.14.0` makes review bundle verification failures easier to diagnose without changing the verification report contract.

## Added

- Manifest schema failures now include field-level explanations such as invalid summary types, unexpected file map keys, and malformed integrity entry fields.
- Review bundle verification still emits the same `review_bundle.manifest.valid` check ID, but the failure message now includes the first manifest schema errors.
- Regression tests assert actionable messages for invalid `summary.releaseReady`, unexpected `files` keys, and invalid integrity byte counts.

## Verification

```bash
npm test
npm run release-check
npm publish --access public
```
