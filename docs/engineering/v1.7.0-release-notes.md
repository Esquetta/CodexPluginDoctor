# v1.7.0 Release Notes

`v1.7.0` adds review bundle diffs so maintainers can compare two handoff directories before approving a release or pull request.

## Highlights

- `doctor review-bundle diff --before <dir> --after <dir>` compares two review bundle directories.
- Diff reports include status, runtime policy, release readiness, attestation, release evidence, runtime plan digest, and embedded release evidence changes.
- Risk-increasing changes are promoted to `warn` or `fail` so CI can catch degraded handoffs.
- `doctor.review.bundle.diff` is now part of `doctor contract --json`.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor review-bundle examples/codex-doctor-runtime --output review-bundle-before --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged
node dist/cli.js doctor review-bundle examples/codex-doctor-runtime --output review-bundle-after --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged
node dist/cli.js doctor review-bundle diff --before review-bundle-before --after review-bundle-after --json
```

Use review bundle diffs when comparing release candidates, pull request artifacts, or agent-workbench handoffs.
