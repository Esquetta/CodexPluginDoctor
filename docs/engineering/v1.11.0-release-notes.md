# v1.11.0 Release Notes

`v1.11.0` hardens review bundle verification by confining manifest-declared artifact paths to the bundle directory.

## Highlights

- `doctor review-bundle verify` now fails when a declared bundle file path resolves outside the review bundle directory.
- Integrity entries must point to the same path declared in the bundle manifest for that file key.
- Regression tests cover escaped manifest paths and mismatched integrity paths.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
npm run release-check
```

Use this release when review bundles are shared through CI artifacts or external reviewer handoffs.
