# v1.6.0 Release Notes

`v1.6.0` adds offline verification for review bundles so generated handoff directories can be checked before reviewers trust them.

## Highlights

- `doctor review-bundle verify <bundle-dir> --target <path> --sign-key-env NAME` verifies a review bundle directory.
- Verification checks the bundle manifest, expected files, runtime plan, runtime policy, signed attestation, and signed release evidence.
- Tampered release evidence or attestation data causes a failed verification report.
- `doctor.review.bundle.verification` is now part of `doctor contract --json`.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor review-bundle examples/codex-doctor-runtime --output review-bundle --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged
node dist/cli.js doctor review-bundle verify review-bundle --target examples/codex-doctor-runtime --sign-key-env DOCTOR_SIGNING_KEY --json
```

Use review bundle verification before attaching bundle artifacts to pull requests, GitHub Releases, or external agent-workbench reviews.
