# v1.23.0 Release Notes

`v1.23.0` hardens MCP security auditing around package-external runtime inputs.

## Added

- `plugin.security.path_traversal_risk` flags path-like MCP runtime arguments such as `--config`, `--script`, `--cwd`, `--require`, and `--loader` when they resolve outside the plugin package root.
- `plugin.security.dangerous_env_usage` flags code-loading environment variables such as `NODE_OPTIONS`, `NODE_PATH`, `PYTHONPATH`, `LD_PRELOAD`, and `DYLD_INSERT_LIBRARIES` when they can inject or load code outside the reviewed package surface.
- The public rule catalog documents both new security finding IDs.

## Compatibility

The new checks are additive security failures. They do not change existing JSON schema shapes or rename existing finding IDs.

## Verification
