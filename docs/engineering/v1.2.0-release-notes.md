# v1.2.0 Release Notes

`v1.2.0` adds runtime approval planning so teams can review local MCP execution before enabling runtime probes.

## Highlights

- `doctor runtime-plan <path>` generates a non-executing runtime plan with MCP server command, args, cwd, probe methods, risk reasons, and a stable approval digest.
- `check --runtime --require-runtime-approval --runtime-approval-digest <digest>` refuses to start runtime probes unless the current plan matches the approved digest.
- Release evidence now records `runtimeApproval` status and includes that data in the signed evidence payload.
- `doctor.runtime.plan` is now part of `doctor contract --json`.
- Security docs now explicitly distinguish runtime approval gates from hard OS or container sandboxing.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor runtime-plan examples/codex-doctor-runtime --json --output runtime-plan.json
node dist/cli.js check examples/codex-doctor-runtime --runtime --require-runtime-approval --runtime-approval-digest <digest-from-runtime-plan> --json
```

Use `doctor runtime-plan` before CI or release runtime probing when local MCP server execution requires explicit review.
