# v1.10.0 Release Notes

`v1.10.0` tightens review bundle integrity verification so reviewers can trust that the manifest covers exactly the expected bundle files.

## Highlights

- `doctor review-bundle verify` now fails when a manifest integrity entry is missing for an expected bundled file.
- Verification now fails when the manifest contains unexpected integrity entries outside the declared bundle files.
- Older review bundles without an integrity block remain backward-compatible.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
npm run release-check
```

Use this release when review bundles are stored as CI artifacts or shared with external reviewers.
