# v1.8.0 Release Notes

`v1.8.0` brings review bundle handoffs into the GitHub Action so CI can upload signed review artifacts alongside JSON, Markdown, and SARIF reports.

## Highlights

- The GitHub Action now supports `review-bundle: "true"` to generate a signed `review-bundle/` directory inside `output-dir`.
- `review-bundle-verify: "true"` writes `review-bundle-verification.json` for offline verification evidence.
- New action outputs expose `review-bundle-path` and `review-bundle-verification-path`.
- `doctor review-bundle verify` now supports `--output <path>` for CI artifact persistence.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
npm run release-check
```

For GitHub Actions, provide the signing key through an environment variable such as `CODEX_PLUGIN_DOCTOR_SIGNING_KEY` and set `signing-key-env` to that variable name.
