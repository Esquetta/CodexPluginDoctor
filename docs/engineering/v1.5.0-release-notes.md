# v1.5.0 Release Notes

`v1.5.0` adds review bundles so maintainers can hand off all approval and release evidence in one directory.

## Highlights

- `doctor review-bundle <path> --output <dir> --sign-key-env NAME` writes a complete local review bundle.
- Bundles include `summary.md`, `manifest.json`, `runtime-plan.json`, `runtime-plan.md`, `runtime-policy.json`, `runtime-policy.txt`, `attestation.json`, and `release-evidence.json`.
- Review bundles reuse signed attestation and release evidence flows, so signing keys stay environment-only.
- `doctor.review.bundle` is now part of `doctor contract --json`.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor review-bundle examples/codex-doctor-runtime --output review-bundle --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged
node dist/cli.js doctor review-bundle examples/codex-doctor-runtime --output review-bundle --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged --json
```

Use review bundles for pull request artifacts, GitHub Release evidence, or external agent-workbench review flows.
