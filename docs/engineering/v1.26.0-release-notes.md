# v1.26.0 Release Notes

`v1.26.0` makes deterministic finding fingerprints instance-specific across validation, runtime, trust, security, and generic MCP reports.

## Added

- Manifest findings identify the package-relative manifest path and affected field.
- Skill findings identify the skill, `SKILL.md`, frontmatter field, or missing support asset.
- MCP findings identify the package-relative config, server, and affected field.
- Runtime findings identify the server and MCP method, plus selected tool, resource, or prompt where applicable.
- Oversized runtime warnings expose only the measured content length, not response content.
- Trust findings identify lifecycle scripts, dependencies, dependency sections, and version specs.

## Security

Evidence does not include secrets, stderr, prompt bodies, resource contents, tool result payloads, or encoded command payloads.

Missing-manifest security audits now use the expected package-relative manifest path instead of an absolute machine path.

## Compatibility

The fingerprint algorithm, rule IDs, severities, schema versions, and command behavior are unchanged. This release only populates the existing optional `evidence` field more consistently.

Package-wide findings may retain one rule-level identity when no narrower instance exists.
