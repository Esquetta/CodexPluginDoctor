# v1.27.0 Release Notes

`v1.27.0` adds targeted, auditable suppression of individual findings without disabling an entire rule.

## Added

- `.codex-doctor.json` accepts `suppressions` with an exact finding fingerprint, review reason, and expiration date.
- Active suppressions move only the matching instance to `suppressedFindings`.
- Text and Markdown reports show suppression counts and audit details.
- JSON reports expose `suppressedFindings` and `suppressionSummary`.
- SARIF keeps suppressed findings out of active results and records them in run properties.
- `suppression.invalid` and `suppression.expired` warnings expose configuration problems.

## Safety

Invalid and expired records never hide findings. Suppression matching requires the exact lowercase SHA-256 fingerprint, and duplicate active records use the first valid match.

Existing `ignoreRules` behavior is unchanged for users who intentionally disable an entire rule.
