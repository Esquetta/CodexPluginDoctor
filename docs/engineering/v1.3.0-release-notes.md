# v1.3.0 Release Notes

`v1.3.0` turns runtime approval plans into review-ready artifacts that can be attached to pull requests, release evidence, or agent workbench transcripts.

## Highlights

- `doctor runtime-plan <path> --markdown` renders a human-readable runtime review plan.
- Markdown artifacts include intent, execution boundary, approval digest, reviewer checklist, server commands, probe methods, and risk reasons.
- `--output <path>` writes Markdown when `--markdown` is selected; existing JSON output behavior remains unchanged.
- JSON runtime plan digests remain stable and unchanged by Markdown rendering.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor runtime-plan examples/codex-doctor-runtime --markdown --output runtime-plan.md
node dist/cli.js doctor runtime-plan examples/codex-doctor-runtime --json --output runtime-plan.json
```

Use the Markdown artifact when a reviewer needs to approve local MCP runtime probing before CI or release execution.
