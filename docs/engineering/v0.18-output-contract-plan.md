# v0.18 Output Contract Plan

## Goal

Publish a machine-readable output contract so CI users and package authors can depend on Codex Plugin Doctor JSON surfaces before the 1.0 release.

This is the first stabilization pass for:

- JSON schema discovery
- schema version visibility
- rule catalog freeze metadata
- rule catalog digest verification

## Command

```bash
codex-plugin-doctor doctor contract [--json] [--output <path>]
```

## Contract Artifact

The command should emit:

- `schemaVersion`
- `kind`
- `generatedAt`
- `version`
- `contract`
- `ruleCatalog`
- `schemas`

The `schemas` list should cover the public JSON outputs that users are most likely to automate against:

- `check --json`
- `security --json`
- `compat --json`
- `mcp --json`
- `audit --json`
- `fix --dry-run --json`
- `history --json`
- `doctor --json`
- `doctor recommend --json`
- `doctor trust --json`
- `doctor perf --json`
- `doctor export --bundle --json`
- `doctor attest --json`
- `doctor npm --json`
- `doctor diff --json`
- `doctor inspector --json`
- `doctor snapshot --json`

## Rule Freeze

The contract should expose a deterministic digest over rule IDs, categories, and default severities. The digest gives downstream tooling a cheap way to detect rule catalog changes between CLI versions.

`0.18.0` freezes the existing rule ID shape for the 1.0 path. New rules may still be added before 1.0, but renaming existing rule IDs or changing default severities should be treated as a breaking release decision.

## Verification

- Add tests that rule IDs are unique and the freeze digest is deterministic.
