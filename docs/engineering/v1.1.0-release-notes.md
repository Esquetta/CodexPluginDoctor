# v1.1.0 Release Notes

`v1.1.0` turns release evidence from a generated report into a verifiable release artifact workflow.

## Highlights

- `doctor release-evidence verify <evidence.json> --target <path> --sign-key-env NAME` verifies shared release evidence offline against an explicit package path.
- Release evidence now includes a top-level `evidenceSignature` so tampering with release readiness, git gates, package metadata, or embedded evidence sections is detected.
- `doctor release-evidence asset <path> --tag <tag> --output <evidence.json> --sign-key-env NAME` writes a signed evidence JSON file and prints the matching `gh release upload ... --clobber` command.
- Add `--upload` to `doctor release-evidence asset` to upload the evidence file to a GitHub Release through the GitHub CLI.
- `doctor.release.evidence.verification` and `doctor.release.evidence.asset` are now part of `doctor contract --json`.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
$env:DOCTOR_SIGNING_KEY = "local-release-secret"
node dist/cli.js doctor release-evidence asset examples/codex-doctor-runtime --tag v1.1.0 --output release-evidence.json --json --sign-key-env DOCTOR_SIGNING_KEY --allow-dirty --allow-untagged
node dist/cli.js doctor release-evidence verify release-evidence.json --target examples/codex-doctor-runtime --json --sign-key-env DOCTOR_SIGNING_KEY
```

For final release evidence after tagging, omit `--allow-dirty` and `--allow-untagged` so strict git release gates must pass.
