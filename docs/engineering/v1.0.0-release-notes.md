# v1.0.0 Release Notes

`v1.0.0` promotes Codex Plugin Doctor from the `0.x` readiness line to a stable public baseline for local Codex plugin validation, npm-distributed CLI usage, and GitHub Action release gates.

## Highlights

- Stable CLI package distributed as `codex-plugin-doctor@latest`.
- Stable GitHub Action pin available as `Esquetta/CodexPluginDoctor@v1.0.0`.
- Public output contract remains at `schemaVersion: "1.0.0"` with `stable-through-1.0` compatibility metadata.
- Validation corpus, runtime probing, security scorecard, trust score, compatibility matrix, npm preinstall scan, local attestation, recommendations, diffing, performance profiling, and export bundle surfaces are included in the stable baseline.
- GitHub Action reports can emit JSON, Markdown, SARIF, step summaries, and uploaded artifacts.

## Compatibility

- Existing finding IDs and default severities are treated as stable through the 1.0 line.
- Runtime probing remains opt-in because it can start package-local MCP servers.
- Public JSON report semantics should stay backward-compatible for 1.x releases.
- Future MCP Doctor expansion is post-1.0 scope; the product remains Codex-first for this release.

## Known Limits

- The CLI validates package quality and local runtime behavior; it does not certify marketplace approval.
- Attestations are deterministic local artifacts but are not signed.
- Consumers own GitHub artifact retention and Code Scanning upload policy.
- `doctor npm codex-plugin-doctor@latest` is expected to report `plugin.manifest.missing` because the CLI package itself is not a Codex plugin package.

## Validation

Before publishing, run:

```bash
npm run release-check
node dist/cli.js self-test
node dist/cli.js doctor corpus
node dist/cli.js doctor contract --json --output output-contract.json
node dist/cli.js check examples/codex-doctor-runtime --runtime --no-animations
```

After publishing, run:

```bash
npm install -g codex-plugin-doctor@latest
codex-plugin-doctor --version
codex-plugin-doctor self-test
codex-plugin-doctor doctor corpus
npm run verify-release-sync
```
