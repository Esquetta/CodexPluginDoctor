# v0.2.0 Release Notes

Release date: 2026-04-30

## Summary

`v0.2.0` expands Codex Plugin Doctor from a Codex-only preflight checker into the first version of a broader MCP compatibility tool.

The release adds a compatibility matrix command that can evaluate a package against Codex and Generic MCP expectations, while reserving Claude Desktop and Cursor as planned client adapters.

## Highlights

- Added `codex-plugin-doctor compat <path>` for client compatibility checks.
- Added Codex and Generic MCP compatibility rows.
- Added skipped placeholders for Claude Desktop and Cursor adapters.
- Added standalone `.mcp.json` package support, so Generic MCP packages no longer need a Codex plugin manifest.
- Added `compat --json` and `compat --json --output <path>` for CI and automation.
- Added `compat --client <client>` filtering for focused checks.

## Example Commands

```bash
codex-plugin-doctor compat .
codex-plugin-doctor compat . --client codex
codex-plugin-doctor compat . --client generic-mcp
codex-plugin-doctor compat . --json
codex-plugin-doctor compat . --json --output compatibility.json
```

## Compatibility Clients

| Client | Status in v0.2.0 | Notes |
| --- | --- | --- |
| Codex | Implemented | Uses the existing Codex plugin validation pipeline. |
| Generic MCP | Implemented | Validates `.mcp.json` package compatibility. |
