# v1.9.0 Release Notes

`v1.9.0` hardens review bundles with manifest-level file integrity so reviewers and CI can detect tampered bundle files before trusting a handoff.

## Highlights

- Newly generated review bundle manifests now include SHA-256 digests and byte sizes for bundle files.
- `doctor review-bundle verify` recomputes those digests and fails when a bundled file no longer matches the manifest.
- Verification remains backward-compatible with older review bundles that do not include an integrity block.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
npm run release-check
```

Use this release when review bundles are shared as CI artifacts, pull request artifacts, or external reviewer handoffs.
