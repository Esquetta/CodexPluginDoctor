# v0.1.2 Release Notes

## Summary

`v0.1.2` clarifies the first-run npm experience.

The CLI correctly fails when run against the Codex Plugin Doctor source repository because that repository is not itself a Codex plugin package. The README now makes that explicit and points users at plugin package directories.

## Release Status

- Version: `0.1.2`
- Status: final
- Package publish: public npm
- Distribution: npm registry, GitHub repository, GitHub Releases, local `npm pack`, local `npm link`
- License: MIT

## Changes Since `v0.1.1`

- clarified that `codex-plugin-doctor check .` expects a Codex plugin package root containing `.codex-plugin/plugin.json`
- changed npm quick start examples to use `path/to/plugin-package`
- added a repository self-test command using `examples/codex-doctor-runtime`

## Install

```bash
npm install -g codex-plugin-doctor
codex-plugin-doctor check path/to/plugin-package
```

## Self-Test From This Repository

```bash
codex-plugin-doctor check examples/codex-doctor-runtime --runtime --no-animations
```
