# v0.1.4 Release Notes

## Summary

`v0.1.4` removes the need for users to know local Codex plugin filesystem paths before running validation.

The CLI can now discover installed Codex plugins from the local Codex home directory and validate them directly from the plugin cache. This makes the first local workflow practical for users who already have Codex installed but do not know where plugin packages are stored.

## Release Status

- Version: `0.1.4`
- Status: final
- Package publish: public npm
- Distribution: npm registry, GitHub repository, GitHub Releases, local `npm pack`, local `npm link`
- License: MIT

## Changes Since `v0.1.3`

- added `codex-plugin-doctor --version`
- added `codex-plugin-doctor list --installed`
- added `codex-plugin-doctor check --installed`
- added `codex-plugin-doctor check --installed <filter>`
- added automatic installed plugin discovery from `CODEX_HOME`, or `~/.codex` when `CODEX_HOME` is not set
- documented installed plugin discovery in the README

## Install

```bash
npm install -g codex-plugin-doctor
codex-plugin-doctor --version
```

## Validate Installed Codex Plugins

```bash
codex-plugin-doctor list --installed
codex-plugin-doctor check --installed
codex-plugin-doctor check --installed github
```

## Verification

Run before publishing:

```bash
npm run prepare-release
node dist/cli.js --version
node dist/cli.js list --installed
node dist/cli.js check --installed github --no-animations --ascii
```

Run after publishing:

```bash
npm view codex-plugin-doctor version
npm install -g codex-plugin-doctor@latest
codex-plugin-doctor --version
codex-plugin-doctor list --installed
```
