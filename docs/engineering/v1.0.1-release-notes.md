# v1.0.1 Release Notes

`v1.0.1` is an additive post-1.0 release focused on release evidence, local signing, MCP package coverage, and CI-friendly performance budgets.

## Highlights

- Local attestations can now be signed with HMAC-SHA256 via `doctor attest --sign-key-env NAME`.
- `doctor perf` can fail when total or per-stage runtime budgets are exceeded.
- The validation corpus now includes a bundled generic MCP package in addition to Codex plugin examples.
- `doctor mcp <path>` exposes the static generic MCP health report under the doctor command family.

## Compatibility

- Existing unsigned attestation artifacts remain supported.
- Runtime probing remains opt-in; `doctor mcp` does not start local MCP servers.
- Public JSON schema surfaces continue to use `schemaVersion: "1.0.0"` for backward-compatible additive fields.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor corpus
node dist/cli.js doctor attest examples/codex-doctor-runtime --json --sign-key-env CODEX_PLUGIN_DOCTOR_SIGNING_KEY
node dist/cli.js doctor perf examples/codex-doctor-runtime --max-total-ms 5000
node dist/cli.js doctor mcp examples/codex-doctor-generic-mcp --json
```
