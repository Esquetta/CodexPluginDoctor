# v0.1.5 Release Notes

## Summary

`v0.1.5` expands Codex Plugin Doctor from a local validator into a more complete workflow tool for plugin authors and CI.

This release adds rule explanations, a public rule catalog, installed plugin summaries, local policy configuration, SARIF output, a GitHub Action entrypoint, support asset integrity checks, and plugin scaffolding.

## Release Status

- Version: `0.1.5`
- Status: final
- Package publish: public npm
- Distribution: npm registry, GitHub repository, GitHub Releases, local `npm pack`, local `npm link`
- License: MIT

## Changes Since `v0.1.4`

- `codex-plugin-doctor explain <finding-id>`
- rule catalog documentation under `docs/rules/catalog.md`
- `codex-plugin-doctor check --installed --all-summary`
- `.codex-doctor.json` support for `ignoreRules` and `failOnWarnings`
- `--sarif` output for GitHub Code Scanning compatible reports
- root `action.yml` composite GitHub Action entrypoint
- skill support asset integrity warnings for backticked `scripts/`, `templates/`, `assets/`, and `examples/` references
- `codex-plugin-doctor init [path]` plugin package scaffolding

## Install

```bash
npm install -g codex-plugin-doctor@latest
codex-plugin-doctor --version
```

## New Commands

```bash
codex-plugin-doctor explain plugin.manifest.missing
codex-plugin-doctor check --installed --all-summary
codex-plugin-doctor check . --sarif --output results.sarif
codex-plugin-doctor init my-plugin
```

## GitHub Action

```yaml
- uses: Esquetta/CodexPluginDoctor@v0.1.5
  with:
    path: .
    runtime: "false"
```
