# v1.24.0 Release Notes

`v1.24.0` adds structured finding evidence so humans and CI systems can see exactly why a finding was raised.

## Added

- Findings may now include an optional `evidence` object in JSON-compatible reports.
- MCP security findings include machine-readable evidence such as server name, MCP config path, env key, arg flag, arg value, and resolved escaped path where applicable.
- Text, Markdown, and security scorecard output render a compact `Evidence:` line when evidence is present.
- SARIF results carry finding evidence under `properties.evidence` for downstream security tooling.

## Security

Hard-coded secret findings redact the secret value in evidence and only expose the env key plus `[REDACTED]` marker.

## Compatibility

The `evidence` field is optional and additive. Existing finding IDs, severities, and report shapes remain backward compatible.
