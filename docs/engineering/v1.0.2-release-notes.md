# v1.0.2 Release Notes

`v1.0.2` completes the signed attestation loop by adding offline verification for artifacts created by `doctor attest`.

## Highlights

- `doctor attest verify <attestation.json> --target <path> --sign-key-env NAME` verifies package fingerprint, report digest, and HMAC-SHA256 signature checks.
- Verification keys are env-only; inline verification keys are rejected to avoid leaking secrets through shell history or process arguments.
- Verification reports expose `kind: "doctor.attestation.verification"` for automation.
- Signed digest generation is stable across copied package roots by excluding path-only recommendation command text from the report digest.

## Security Notes

The signed payload covers stable evidence fields: subject, package fingerprint, report digest, summary, schema version, and attestation version.

The verifier explicitly reports these display fields as unsigned metadata:

- `generatedAt`
- `targetPath`
- `verification`
- `signature.keyHint`

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
$env:CODEX_PLUGIN_DOCTOR_SIGNING_KEY = "local-test-secret"
node dist/cli.js doctor attest examples/codex-doctor-runtime --json --sign-key-env CODEX_PLUGIN_DOCTOR_SIGNING_KEY --output attestation.json
node dist/cli.js doctor attest verify attestation.json --target examples/codex-doctor-runtime --json --sign-key-env CODEX_PLUGIN_DOCTOR_SIGNING_KEY
```
