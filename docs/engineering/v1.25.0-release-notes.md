# v1.25.0 Release Notes

`v1.25.0` adds deterministic finding fingerprints so repeated diagnostics can be tracked across local runs, CI jobs, machines, and report formats.

## Added

- Findings emitted by validation, runtime probing, security audits, trust scoring, and generic MCP checks now include an optional `fingerprint`.
- Fingerprints use SHA-256 over a versioned payload containing the rule ID and canonical structured evidence.
- Path evidence is normalized to `/` and made relative to the analyzed package root when possible.
- Text, Markdown, security scorecard, trust, and generic MCP reports render a compact `Fingerprint:` line.
- SARIF results expose the same identity through `partialFingerprints["codexPluginDoctor/v1"]`.

## Security

Fingerprint generation uses only evidence already present in the finding. Redacted values remain `[REDACTED]`; hidden secret values are not read or hashed.

## Compatibility

The `fingerprint` field is optional and additive. Existing finding IDs, severities, schema versions, and command behavior remain backward compatible.

Findings without structured evidence use a stable rule-level fingerprint. Future targeted suppression or baseline features can add more instance-specific evidence without changing this release's CLI behavior.
