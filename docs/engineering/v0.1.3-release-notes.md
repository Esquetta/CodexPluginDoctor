# v0.1.3 Release Notes

## Summary

`v0.1.3` improves first-run guidance when users run `codex-plugin-doctor check .` from the wrong directory.

Instead of returning only the generic missing-manifest failure, the CLI now distinguishes between the Codex Plugin Doctor source repository and ordinary non-plugin Node projects. This keeps the failure strict while making the next command obvious.

## Release Status

- Version: `0.1.3`
- Status: final
- Package publish: public npm
- Distribution: npm registry, GitHub repository, GitHub Releases, local `npm pack`, local `npm link`
- License: MIT

## Changes Since `v0.1.2`

- added source-repository-aware guidance for `plugin.manifest.missing`
- added a self-test command suggestion when the Codex Plugin Doctor source repo is checked directly
- added clearer guidance for normal Node projects that are not Codex plugin package roots
- added regression coverage for source-repo and non-plugin-project missing-manifest messages

## Install

```bash
npm install -g codex-plugin-doctor
codex-plugin-doctor check path/to/plugin-package
```

## Self-Test From This Repository

```bash
codex-plugin-doctor check examples/codex-doctor-runtime --runtime --no-animations
```

## Verification

Run before publishing:

```bash
npm run prepare-release
node dist/cli.js check . --no-animations --ascii
node dist/cli.js check examples/codex-doctor-runtime --runtime --no-animations --ascii
```

```bash
npm view codex-plugin-doctor version
```
