# v0.19 Validation Corpus Plan

## Goal

Add a built-in validation corpus that exercises realistic package shapes before releases. The corpus should make release confidence less dependent on isolated unit fixtures and more grounded in bundled examples that ship with the npm package.

## Command

```bash
codex-plugin-doctor doctor corpus [--json] [--output <path>]
```

## Corpus Cases

The first corpus wave should use package examples already shipped in the npm artifact:

- `bundled-runtime-healthy`: a runtime-complete package expected to pass static and runtime checks
- `bundled-risky-security`: a risky package expected to fail with the hard-coded secret finding
- `bundled-starter-skill`: a minimal skill-only package expected to pass static validation

## Report Contract

The command should emit:

- `schemaVersion`
- `kind`
- `generatedAt`
- `version`
- `summary`
- `cases`

Each case should include source type, profile, expected status, actual status, finding IDs, and whether expectations matched.

## Safety

- Use only bundled example packages by default.
- Runtime probing is limited to the known runtime sample.
- Do not fetch external packages or execute npm lifecycle scripts.
- Keep JSON output deterministic enough for CI dashboards.

## Verification

- Add focused tests for JSON output, text output, output-file writing, missing output path handling, and expectation mismatch behavior.
- Run `npm test`.
- Run `npm run build`.
- Run `npm run release-check`.
