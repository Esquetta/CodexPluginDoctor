# v0.1.0 Release Notes

## Summary

`v0.1.0` is the first open-source-ready local release of Codex Plugin Doctor.

It packages the CLI validator, runtime MCP protocol probe, CI-friendly reporting flow, and real-world validation evidence into a repository that can be made public without needing a separate website.

## Release Status

- Version: `0.1.0`
- Status: final
- Package publish: not yet public npm
- Distribution: GitHub repository, GitHub Releases, local `npm pack`, local `npm link`
- License: MIT

## Highlights

- Validates Codex plugin manifests, skill metadata, and `.mcp.json` package wiring.
- Detects path traversal and hard-coded secret-like environment values.
- Runs real MCP stdio protocol checks behind `--runtime`.
- Validates `initialize`, `tools/list`, `tools/call`, `resources/list`, `resources/read`, `resources/templates/list`, `prompts/list`, and `prompts/get`.
- Supports JSON, Markdown, and human text reports.
- Keeps machine-readable output deterministic while rendering interactive status to `stderr`.
- Includes redacted verbose runtime transcripts.
- Emits runtime capability scorecards.
- Provides CI artifact generation and release-candidate workflows.
- Ships MIT license, contribution docs, security policy, issue templates, and funding config.

## Real-World Validation Evidence

Validation sessions now cover:

- repo-owned examples and fixtures
- curated Codex plugin cache packages
- marketplace-style plugin snapshots
- YAML block scalar skill descriptions
- media and visual workflow metadata

These waves reduced known false positives and fixed one parser-level false negative for block scalar frontmatter descriptions.

## Install and Run Locally

```bash
npm install
npm run build
node dist/cli.js check examples/codex-doctor-runtime --runtime
```

For local global testing:

```bash
npm link
codex-plugin-doctor check examples/codex-doctor-runtime --runtime
```

## Verification

Run:

```bash
npm test
npm run build
npm run prepare-release
node dist/cli.js check examples/codex-doctor-runtime --json --runtime
node dist/cli.js check examples/codex-doctor-risky --ascii
```

Expected:

- all tests pass
- TypeScript build succeeds
- `npm pack --dry-run` includes `LICENSE`, `README.md`, `package.json`, and `dist`
- runtime example reports `PASS`
- risky example reports expected hard-coded-secret `FAIL`

## Known Limits

- Public npm publish is intentionally deferred.
- Runtime probing currently focuses on command-based MCP servers.
- OAuth/browser authentication flows are out of scope for `v0.1.0`.
- Hosted dashboard, docs website, and web report viewer are out of scope.
- Description heuristics are evidence-tuned but should continue to use validation-session notes for future changes.
