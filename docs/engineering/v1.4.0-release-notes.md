# v1.4.0 Release Notes

`v1.4.0` adds a runtime policy advisor so operators can decide how to handle local MCP execution before starting runtime probes.

## Highlights

- `doctor runtime-policy <path>` classifies runtime execution as `allow`, `review`, `sandbox_recommended`, or `deny`.
- JSON output is available with `--json` and is published as `doctor.runtime.policy` in the output contract.
- The advisor reuses runtime plan and security findings, including approval digests, server risk levels, and deny-level command-surface risks.
- Text output gives a concise decision, reason, recommended actions, and per-server policy classification.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
node dist/cli.js doctor runtime-policy examples/codex-doctor-runtime
node dist/cli.js doctor runtime-policy examples/codex-doctor-runtime --json --output runtime-policy.json
node dist/cli.js doctor runtime-policy examples/codex-doctor-risky --json
```

Use `runtime-policy` before `runtime-plan` approval when a reviewer needs a quick answer to whether local MCP execution should be allowed, reviewed, sandboxed, or denied.
