# v1.12.0 Release Notes

`v1.12.0` closes symbolic-link and junction escape paths during review bundle verification.

## Highlights

- Review bundle artifact paths are resolved with their canonical filesystem locations before being trusted.
- Files reached through a symbolic link or Windows junction are rejected when their canonical location is outside the bundle directory.
- The protection applies to manifest, runtime plan, runtime policy, attestation, release evidence, and integrity reads.
- Regression coverage uses a valid external runtime plan to prove that content and digest validity cannot bypass path confinement.

## Verification

Run these checks before publishing:

```bash
npm test
npm run build
npm run release-check
```

Use this release when review bundles may come from CI artifacts or other untrusted handoff channels.
