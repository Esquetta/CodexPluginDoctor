# v1.13.0 Release Notes

`v1.13.0` tightens review bundle verification by validating the manifest schema before trusting bundled metadata.

## Added

- Review bundle verification now rejects malformed manifest summaries, invalid status and runtime policy values, invalid exit codes, unexpected bundle file keys, and malformed integrity entries.
- Integrity entries now require `sha256:<hex>` digests and non-negative safe integer byte counts.
- Regression tests cover invalid summary fields, unexpected file map keys, and invalid integrity entry shapes.

## Verification

```bash
npm test
npm run release-check
npm publish --access public
```
