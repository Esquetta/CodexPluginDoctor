# Brand Foundation

## Brand Purpose

Codex Plugin Doctor exists to make plugin quality legible. It turns hidden packaging and runtime problems into visible, fixable engineering work.

## Brand Promise

`Know if it works before anyone else has to find out.`

## Brand Position

Codex Plugin Doctor is a precision tool for builders who care about trust, release quality, and professional developer experience.

## Brand Attributes

- precise
- technical
- trustworthy
- practical
- calm under failure

## Brand Personality

### We Are

- direct
- rigorous
- credible
- operationally aware

### We Are Not

- flashy
- vague
- playful about production risk
- overloaded with AI hype language

## Tagline Options

- Validate before release.
- Ship Codex plugins with confidence.
- Package truth, before production.

## Recommended Tagline

`Ship Codex plugins with confidence.`

## Naming Rationale

The word `Doctor` communicates diagnosis, triage, and remediation. It fits a validation product that must explain failures clearly without sounding academic or enterprise-heavy.

## Visual Direction

### Design Theme

Clean technical confidence with light diagnostic cues.

### Color Palette

- Primary: `#0E1A2B`
- Accent: `#0FA3B1`
- Success: `#2F855A`
- Warning: `#D69E2E`
- Failure: `#C53030`
- Neutral background: `#F6F8FB`

### Typography Direction

- Headlines: `Space Grotesk` or `Sora`
- Body: `IBM Plex Sans` or `Inter Tight`
- Code and CLI surfaces: `IBM Plex Mono`

## Voice Guidelines

- lead with facts
- explain impact clearly
- avoid hype adjectives
- make remediation sound achievable
- speak like a senior developer tool company, not a consumer startup
