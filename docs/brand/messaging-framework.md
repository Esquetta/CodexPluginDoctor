# Messaging Framework

## Core Narrative

Codex Plugin Doctor helps plugin authors and engineering teams detect package, runtime, and security issues before release. It replaces guesswork and support-driven validation with deterministic reports.

## One-Line Value Proposition

Validate Codex plugin packages before they break trust in production.

## Expanded Value Proposition

Codex Plugin Doctor is a CLI-first validation tool for Codex plugins, skills, and MCP integrations. It checks package structure, startup behavior, tool discovery, schema quality, and security-sensitive configuration, then produces an actionable report for local development or CI.

## Messaging Pillars

### 1. Runtime Truth

Static config passing is not enough. The tool probes runtime behavior and surfaces failures users would otherwise find first.

### 2. Release Confidence

Authors can ship with evidence, not assumptions.

### 3. Actionable Reports

Every issue should explain what failed, why it matters, and what to fix next.

## Proof Statements
